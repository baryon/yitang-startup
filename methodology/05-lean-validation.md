# 低成本验证 — 三种方式回答关键假设

## 核心公式

```
验证成本 = 假设数量 × 验证方式 × 资源投入
```

每跨一个验证方式，成本就是一个量级的提升：

```
常识  ──→  调研  ──→  测试  ──→  全量开发
¥0       ¥几千    ¥几万    ¥百万+
```

> 99% 的创业失败，是用最重的方式验证最早期的假设。
> 用对方法，可以用 1/10 甚至 1/100 的成本拿到同样的认知。

## 方式 1：靠常识（行业经验值 / Benchmark）

### 什么是 Benchmark
基准值 / 经验值。**很多答案行业里已有共识，问对人就有。**

### 常见 Benchmark 例子
- 现在职业教育，9.9 体验课在抖音的获客成本范围？
- 零售业务毛利率超过多少，才能做网红直播？
- 流量平台一个 DAU 大致提升多少公司估值？
- 线下零售毛利低于多少，建不起来分销渠道？
- 餐饮业务，租金占营收多少算健康？
- SaaS 行业，LTV / CAC 比值多少算良好？

### 怎么拿到
1. 找行业老兵深度访谈（1-2 小时一杯咖啡）
2. 找做过同领域的 FA / 投资人
3. 上你所在赛道的"行业失败案例库"
4. 找做过这事的同学问

### 适用场景
- 行业已成熟，有历史数据
- 你是新手入行，缺基本常识
- 想快速排除 30% 明显不可行的方向

### 一堂提醒
> 相信常识最重要。
> 创了半天业，最后发现书里都有。

## 方式 2：靠调研（竞争对手 + 用户访谈）

### 三大调研误区
- **以为原创** — 大概率别人做过
- **盲目自信** — 不查真实情况
- **闭门造车** — 不和真实环境接触

> **没有调查，就没有发言权。**
> **没有调研，就没有决策权。**

### 实战调研手法（一堂学长真实操作）

#### 调研竞争对手
- **入职门店调研**：一周时间数 SKU、找小票、半夜数据分析 → 拿到单店财务报表
- **菜品成本反向工程**：买回来分拆材料、称重、测算成本 → 发现毛利 70%
- **假装合作**：用"假装合作"的方法，拿到对手公司的成本/复制难度/早期冷启动策略
- **自动监控**：监控对标 IP 账号，自动发现"低粉爆款"，加入"每周改写库"

#### 调研专家
- 找做过同业务的老兵 1v1 访谈
- 找 FA / 投资人吐槽过往看过的失败案例
- 找供应链上下游问真实成本结构

#### 调研用户
- 真实用户访谈 5-15 人，问"上一次是怎么解决的、花了多少钱"
- 不要问"你会不会用"（一定说会）
- 而要问"过去 3 个月你为这事花了多少钱"

### 案例：东南亚电商创始人
- 原计划：开发简版淘宝（100 万）
- 调研后发现：用客服人工模拟微信拼团，几万块就能验证选品 + 支付能力两个关键假设
- 节省 90% 验证成本

### 适用场景
- 行业有竞争对手，有可观察样本
- 用户群体可触达
- 想验证"具体怎么做能成"

## 方式 3：靠实验（精益 MVP）

### 精益思维的本质

> **用少量的时间和试错成本，大幅降低系统性风险。**

### MVP（最小可行产品）≠ 一个简版的产品

MVP 是**测量工具**，可以用来测试 5 步法的任何一个环节。

- 测试需求：着陆页 + 注册按钮，看转化率
- 测试解决方案：手工模拟服务，看用户满意度
- 测试商业模式：跑 1 个柜子 / 1 家店 / 1 个销售人头
- 测试增长：投放 1 万元到 1 个渠道，看 CAC
- 测试壁垒：开放 API 看是否有生态接入

### 实验设计三板斧

#### 单一变量
一次只测一件事。同时测 3 件事，结果归因不清。

#### 小步快跑
- 反例：3 个月开发一个版本，期间团队等结果
- 正例：1-3 周开发一个版本，先测最关键的假设

#### 可证伪
设计前先问：什么结果能让这个假设被否定？
没有"证伪线"的实验都是自我安慰。

### 案例：现烤汉堡（陈先敏，仅花 20 万从 0 到 1）

5 个关键假设：是否好吃 / 是否好卖 / 是否能做 / 是否稳定 / 是否盈利

测试顺序（聪明地剥离）：
1. **测工艺**：从朋友借烤炉 + 借朋友的汉堡店加工场地，做出稳定汉堡胚
2. **测口味**：在朋友店里搭配自制酱料，先内部品尝再邀请顾客免费品尝
3. **测购买**：借朋友店小范围试卖 + 上外卖，测定价和复购

5 轮测试后，才正式开第一家店。
现在 100 家门店，疫情期间逆势扩张。

### 案例：游戏核心玩法
- 核心玩法占游戏成败 99%，但开发量只占 20-30%
- 早期做小版本，灌用户进来观察"多少分钟让用户爽 1 次"
- 核心玩法成立 → 等级/关卡/勋章/结算系统都不是问题

### 案例：洗发水卖点测试
- 4 维矩阵：功效 × 成分 × 香味 × 背书 → 排列组合上千种
- 在电商详情页里发布"假产品"测不同卖点组合的转化率
- 找到 sharp 组合：修复烫染 + 鱼子酱 + 香奈儿 No.5 香氛 + 高端品牌背书

## 三个方式的选择

```
你的关键假设 →

  能查到现成答案？
  ├─ 能 → 常识（找老兵 / 找 FA）
  └─ 不能 → 有竞争对手或用户可触达？
            ├─ 有 → 调研（竞争对手深挖 / 用户访谈）
            └─ 没有 → 实验（设计 MVP 测试）
```

> 永远从最便宜的开始。能用常识回答的，绝不调研；能用调研回答的，绝不开发产品。

## 事前验尸（孙子兵法 + 美国心理学家克莱恩）

> 未料胜先料败。

老兵的思考习惯：
- 提前想：这个项目最可能死在哪里？
- 提前想：哪些点最致命？
- 提前想：如何用最低成本避免它死掉？

花几天时间认真回答这几个问题，可以**给自己多几条命**。

## 一句话心法

> **idea 不值钱，背后的关键假设和验证才值钱。**
> **最好的验证，都是小步快跑、低成本试错，避免在一开始就做一个大而全的产品、一次性赌上所有资源。**
