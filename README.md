## 这个 Skill 能做什么

当你对 Claude 说出以下任一意图时，会自动触发：

- "我想创业 / 准备创业 / 想做副业"
- "我有个 idea ..."、"想做一个 [X] 项目"
- "帮我评估这个生意 / 商业模式可不可行"
- "怎么写商业计划书 / BP"
- "帮我画商业画布 / lean canvas / 五步法"
- "我做了 [X]，下一步怎么办 / 怎么验证"
- "想融资，怎么准备"

Claude 会按以下流程引导你：

```
Phase 0  讲清楚"做业务=验证关键假设"  ← 认知对齐，必须先过这一关
   ↓
Phase 1  Q&A 5 步采集（需求 → 解决方案 → 商业模式 → 增长 → 壁垒）
   ↓
Phase 2  提炼 Top 3 关键假设（前置 / 高风险 / 单一变量原则）
   ↓
Phase 3  生成 v0.1 商业计划书
         ├─ 📊 HTML 可视化商业画布
         ├─ 📝 Markdown 表单式计划书（结构化、易迭代）
         └─ 📖 Markdown 叙事式计划书（流畅段落、适合投递）
   ↓
Phase 4  持续调研 + 迭代（v0.2 → v0.3 → ...）
         每完成一轮验证，更新假设状态 ✅ ❌ 🔄
```

## 安装

### 方式 1：用 `npx skills` 一键安装（推荐）

依赖 [vercel-labs/skills](https://github.com/vercel-labs/skills) 工具，无需手动 clone：

```bash
npx skills add baryon/yitang-startup
```

如需指定安装到 Claude Code（默认会让你选）：


```bash
npx skills add baryon/yitang-startup -g -a claude-code
```

## 输出格式选择

| 模板 | 风格 | 适合场景 |

## 一堂方法论核心摘要

为了让你快速感受这个 Skill 背后的方法论密度，下面是几条核心心法：

> **做业务 ≠ 做一个产品出来。做业务 = 验证一组关键假设是否成立。**

> **不要过早复制未经验证的商业模式。** 复制是给错误加杠杆，把错误放大 10 倍、100 倍。

> **拆 >>> 不拆。** 拆得不够好，差别没那么大；拆与不拆，是两种完全不同的做事路径。

> **找到那杆稀缺的旗子，立在市场上。** 一厘米宽，一公里深。

> **成功不能复制，但失败可以避免。** 创业路上，你能保证错误不再出现第二次，就是非常 NB 的创业者了。

> **不要把创业搞成开盲盒。** 放弃旧地图，才能找到新大陆。
