# 单元模型 — 商业模式的算账工具

## 定义

> **单元模型 = 业务可运营、可复制的最小单元。**
>
> 你业务未来的规模化，核心就是复制这个模型。

用单元模型来：
- 判断商业模式是否可行
- 预测公司财务状况
- 判断能不能持续扩张

每个业务通常有 3-7 个重要的单元模型，其中有 1-2 个最关键。

## 常见单元模型类型

| 模型类型 | 适用业务 |
|---------|---------|
| 单用户首单模型 | 流量产品、电商、SaaS 试用 |
| 单用户 LTV 模型（CAC/LTV） | 投放驱动型业务、订阅 SaaS |
| 单订单模型 | 标品电商、外卖 |
| 单 SKU 模型 | 消费品、跨境电商 |
| 单门店模型 | 线下连锁餐饮、零售 |
| 单履约模型 | O2O、上门服务、配送 |
| 单设备/柜子模型 | 共享设备、自动售货机 |
| 单销售人头模型 | 2B 直销、地推团队 |
| 单老师/讲师模型 | 教育、培训、内容 |
| 单城市/单区域模型 | 多城市扩张型业务 |

> **关键：选错单元模型，后面所有分析都白费。**

### 选模型反例

**洗车 O2O**：选了"单订单模型"，每单看起来赚钱 → 真实是单用户 LTV 模型才决定成败（获客成本 + 复购）

**酒店共享空气净化器**：选了"单台机器模型"，每台微微赚钱就快速扩张 2000 台 → 真实是单酒店 BD 成本 + 单酒店维护成本，赔得一塌糊涂

**T 恤销售**：选了"单订单模型"，每单赚钱 → 真实是单用户模型（要算获客 + 复购 + 仓储 + 营销摊销）

## 算账能力 6 段位

| 段位 | 能力 | 典型错误 |
|------|------|---------|
| **L1** | 不算账 | 只谈初心 / 不谈变现 / 不谈数字 |
| **L2** | 只算大账 | 看公司层大数字，看不到单元模型 / 存在盲区 / 选错模型 |
| **L3** | 模型不完整 | 忽略经营成本 / 组织成本 / 财务成本 |
| **L4** | 数不够准 | 缺少统计意义 / 错误类比 / 乐观预期 |
| **L5** | 缺少动态预测 | 忽略规模变化 / 外部变化 / 业务变化 |
| **L6** | 顶级认知 | 完整账 + 完整指标 + 完整预测，且预测比别人都准 |

> 单元模型是一个动态升级的过程。**只反映当下你对业务的认知，不等于人的能力。一旦转型，往往从 L5/L6 会掉到 L2/L3/L4。**

## L3 易丢成本清单（最常见错误）

### 经营成本（业务侧易丢）
- 损耗 / 货损 / 退货
- 折旧 / 设备维护
- BD 成本 / 拓展成本
- 房租 / 水电 / 物业
- 包装 / 物流摊销
- 客服 / 售后

### 组织成本（管理侧易丢）
- 五险一金（基础工资 × 1.2-2.5 才是真实人力成本）
- 销售提成
- 招聘 / 培训 / 学习成本
- 售前支持人工
- 跨城市管理成本
- 团队腐败 / 跑冒滴漏

### 财务成本（财务侧易丢）
- 押金 / 保证金
- 账期 / 资金占用成本
- 坏账率
- 各类税费（增值税 / 关税 / 进口税）
- 资质 / 牌照 / 注册证
- 平台保障金 / 支付手续费

## L4 数据准确度清单

### 缺少统计意义
- 大 B：至少 2-5 个样本
- 小 B：至少 10-20 个样本
- 流量产品：至少万量级
- 周期波动业务：至少覆盖一个完整周期（季节、工作日/周末）

### 错误类比的常见维度
- 地域差异（一线 vs 三四线、北方 vs 南方）
- 用户画像差异（年龄、收入、职业）
- 渠道差异（线上 vs 线下、流量来源不同）
- SKU 差异（同品类不同商品）
- 风口红利差异（早期入场 vs 后期入场）

### 乐观预期 vs 真实数字（行业经验）
- 转化率：你以为 30%，真实可能 5-15%
- 退货率：你以为 15%，真实可能 30-50%
- 货损率：你以为 5%，真实可能 15%
- 退费率：你以为 8%，真实可能 15-20%
- 入住率/上架率：你以为 80%，真实可能 30-50%

## L5 动态预测三类变化

### 规模变化
- 规模经济：单位成本随规模下降（采购、固定成本摊销）
- 规模不经济：管理成本随规模指数上升（咨询、律所、专家型业务）

### 外部变化（三大窗口）
- **竞争窗口**：特定赛道的下场机会
- **流量窗口**：特别渠道的低价获客机会
- **资本窗口**：特定模式的融资拿钱机会

> 新人往往死于常识，老人往往死于趋势。

### 业务变化（主动调整）
- 自动化替代人工
- 渠道切换
- 涨价/降价
- 上管理机制对抗规模不经济

## 公司账 vs 单元模型账（4 象限）

|              | 单元模型成立 | 单元模型不成立 |
|--------------|------------|---------------|
| **公司账成立** | ✅ 可放心扩张 | ⚠️ 公司侥幸盈利但不可复制 |
| **公司账亏损** | 🟢 早期亏损但可复制（值得投资） | ❌ 必须立刻止损 |

**单元模型的本质**：
> 单元模型是从"赔钱出发"的关键假设。
> 单元模型是你不犯大错误的最后一道能力防线。
> 单元模型是你公司层面，最重要的 ROI 模型。

## 不需要算单元模型的情况

> **单元模型是为「复制」做准备的。**

- 如果业务不需要复制 → 没什么价值
- 如果你只想做一家"小而美"的精品店 → 你的勤奋比算账更重要
- 如果你正在做规模复制 → **不算就是致命的**

## 真实案例：某校园 O2O 项目

- 创始人是以太一个应届生
- 趁 O2O 风口快速融到 B 轮
- 在 B 轮时犯了致命错误：没算清单元模型，不知道为什么有些学校赚钱、有些亏钱
- 直接以城市为单位快速铺全国
- 关键成本项没有测算清楚 → 快速进入第四阶段（增长扩张）→ 几个月烧光所有钱

## 一句话心法

> **创业路上，你能保证错误不再出现第二次，就是非常 NB 的创业者了。**
> **成功不能复制，但失败可以避免。**
