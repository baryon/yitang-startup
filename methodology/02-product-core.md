# 产品内核 — 解决方案的最小可验证单位

## 定义

> **产品内核 = 用户愿意选择你的最小解决方案。**
>
> （选择 = 使用 + 付费）

逐字拆解：
- **用户**：站在 *目标用户* 的视角，不是你想给什么
- **愿意选择**：综合决策强到可以击穿用户习惯（替换成本很高）
- **最小**：不是大而全，而是 3-5 个决定性服务特性的最小集合
- **解决方案**：是用户视角的*服务特性*，不是产品经理视角的*功能清单*

## 为什么需要这一步（而不是直接做商业模式）

如果完整测试商业模式（开店、做产品、做服务）太重，**先把核心解决方案抽出来单独打磨**：

- 现烤汉堡店：先打磨口味 → 复购率提升
- 通讯模组（2B）：先打磨"长期稳定"内核 → 销售转化率提升
- 卤味店：反复测试口味 → 复购率提升
- 一堂线下营：3 季度打磨教研流程 → 作业率 + NPS 大幅提升

> 核心：**剥离一个高价值因素先单独测，再回到完整商业模式。**
> 这样更轻、更快、更容易迭代成功。

## 4 个关键词，逐个解读

### 关键词 1：用户

脱离了用户和需求谈"价值高低"，就是耍流氓。

**反例：汉字演化学习动画片**
- 项目同时想满足：4-12 岁孩子学汉字 + 成年人文化自信 + 外国人学中文
- 三个用户群需要的产品形态完全不同（卡通 vs 纪录片 vs 教学）
- 教训：先选定一个具体用户，再设计内核

**反例：智能儿童监护仪（烧 1000 万）**
- 实验室技术做了 18 个月、花了小 1000 万
- 真实测试时发现：孩子几个月后大量翻身，技术解决不了 → 只能关停
- 教训：技术成立 ≠ 用户场景成立

### 关键词 2：愿意选择

**俞军用户价值公式**：
```
用户价值 = (新体验 - 旧体验) - 替换成本
```

替换成本巨大。用户改变习惯的代价很高。**1%、10%、30% 的提升通常不够**——需要 10 倍提升才能击穿习惯。

**Sharp（锐度）这个词**：
- "你这产品不够 sharp"
- "那个 SaaS 最 sharp 的功能是供应链管理"
- 不够 sharp = 软塌塌、击穿不了用户

**反例：简洁版小鹅通**
- 创业者说："小鹅通太复杂，我做个简洁的针对个人理财师的版本"
- 问题：体验提升 10-15% 撬不动用户迁移
- 加之你失去了头部产品的稳定性、生态、服务保障

**类似声音**：
- 程浩：极大提升效率 / 极大降低成本 / 极大提升用户体验
- 颠覆式创新：只有足够强的创新，才有机会成功
- "十倍创新" 应是创业者的追求

### 关键词 3：最小

> **早期产品，内核要清晰，其他要糙。**

资深产品总监的判断：即便资源充足，早期也要糙一点。
**因为东西做得多了，无法判断哪个是真正的内核价值。**

**反例：自助洗头机（功能清单 17 项）**
- 项目列了 9 个已开发 + 8 个研发中的功能（智能洗发水、灭菌、按摩、新闻播报...）
- 用户真正决策时，只关心 2 件事：**私密**（在公共场合敢躺下吗）+ **安全**（万一头发被洗坏）
- 其他所有功能都是优化项，不是决定项

### 关键词 4：解决方案（不是功能）

层级：流程层 < 功能层 < 服务层 < 商业层

> 用户视角的**服务特性**，而不是产品经理视角的**功能清单**。

实现一个"安全"的服务特性，可以：硬件锁 / 海报 + 口号 / 驻场服务人员 / 营造可信赖环境。多种实现手段。

**通讯芯片 2B 案例**：
- 关键内核 = "长期稳定"（客户最关心）
- 实现手段 ≠ 必须做长期服务合同
- 也可以：做品牌 / 大品牌背书 / 上市公司股东等

## 设计心法（最重要的两个问题）

> 长期地、反复地、认真地和团队问 2 个问题：
>
> 1. **拿掉这一条，是不是很多用户就不买单了？** #决定性
> 2. **有了这一条，是不是帮助也不大？** #优化性

好的产品内核都是短小精悍的，都是足够 Sharp 的。

## 6 段位修炼地图

| 段位 | 能力 | 适用场景 |
|------|------|---------|
| **L1** | 没有价值意识 | — |
| **L2** | 价值意识 — 能描述用户和价值 | 任何独立带产品的人 |
| **L3** | **最小意识** — 能逆向工程找到 Top 3-5 决定性要素 | 独立管新业务 |
| **L4** | **评估方法** — 能用关键指标评估内核质量 | 资源有限、希望尽量成功 |
| **L5** | **快速迭代** — 能大规模测试组合内核 | 内核风险高、需要大量迭代 |
| **L6** | **迁移能力** — 能跨领域应用产品内核思维 | 跨界做事、自建流程 |

### L3 训练：逆向工程拆内核

代入用户决策场景，做减法：哪些拿掉就不买了？

**案例：早期美团外卖（2012）的产品内核**
1. 线上下单，食品安全有保障
2. 配送时间在 50 分钟内（不是 20 分钟，那是优化）
3. 配送费在 7 元内
4. 附近有 50 个商家可选

> 50 分钟就够用了；20 分钟当然更好，但那是优化，不是内核。

### L4 训练：用核心指标评估

不同业务对应不同的内核指标：

| 业务类型 | 核心指标 |
|---------|---------|
| 通用模型 | 流量 × **转化率** × 客单价 × **复购率** × **推荐率** |
| 游戏 | 核心玩法的留存率、付费率 |
| 消费品 | 核心卖点的转化率、复购率 |
| 教育 | 完课率、续费率、NPS |
| 短视频 | 完播率、收藏率、转发率 |
| 内容 IP | 抬头率、互动率 |

### L5 训练：组合迭代

游戏案例（智钊贡献）：
- 核心玩法占游戏成败的 99%，但开发量只占 20-30%
- 早期做小版本，灌用户进来观察"多少分钟让用户爽 1 次"
- 核心玩法 sharp → 等级、关卡、勋章、结算等系统都不是问题

洗发水案例（张磊贡献）：
- 4 维卖点矩阵：功效 × 成分 × 香味 × 背书
- 排列组合上千种 → 在电商详情页测假产品页面，看转化率
- 找到 sharp 组合：修复烫染 + 鱼子酱 + 香奈儿 No.5 香氛 + 高端品牌背书

### L6 训练：迁移能力

抛开方法论，理解本质：**产品价值的二八法则**。

应用场景：课程、医疗、直播、教练、装修、婚礼...
关键策略一大半都是"找到那个内核，不折不扣完成它，非内核可以妥协甚至砍掉"。

## 产品内核分层使用

不是 CEO 专属。**任何独立交付服务的人都需要找到自己负责模块的内核**。

**奶茶店 4 层内核**：
- 品牌内核 → 产品内核 → 单店运营内核 → 单杯产品内核

**一堂 3 层内核**：
- 公司业务内核 → 单课程内核 → 单节课交付内核

每一层都有人负责，每一层都问那两个问题。

## 不大不小刚刚好

不要"往大了做总安全"。不同状态的内核：

| 状态 | 例子 |
|------|------|
| **没强内核** | 宠物照片分享 + O2O；智能水龙头 |
| **单一特性（不够）** | 女性版滴滴；低佣金版美团；全金属抗摔手机 |
| **完整产品（太大）** | 全量洗头机；开新店重投入 |
| **完整愿景（更大）** | 八大平台、N 大板块、3 大生态 |
| ✅ **刚刚好** | 3-5 个决定性服务特性的最小组合 |

## 两个常见困扰

**Q1：我的业务一定能找到强内核吗？**
NO。市场上存在强内核的机会本身就不多。"所有伟大的需求和产品，都一定会被错误的方法和错误的时间点试过很多次"——王慧文。

**Q2：内核必须自己创新吗？**
NO。如果创新性弱，市场容量大，**直接分析别人的内核然后复制就好**。早期的知乎、抖音、多数线下连锁、消费品牌都是这样的。

## 一句话心法

> **找到那杆稀缺的旗子，立在市场上，不断加强它，追求"一厘米宽，一公里深"。**
